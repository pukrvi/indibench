The open, lab-neutral frontier benchmark for India.

LLMs and agents, measured on Indian languages and Indian use cases — across text, speech, vision, and agentic tasks — at a difficulty where today's best models score low. That is the point.

इ IndiBench
02
Problem

1.4 billion people.
Zero seats in the frontier benchmark suite.

Frontier AI has benchmarks it respects — HLE for expert knowledge, SWE-bench for engineering. India is absent. Every open Indian benchmark today is:

Saturated

Frontier models already score 56–77% on MILU, BhashaBench, ParamBench — they measure yesterday, not the frontier.

Contaminated

Scraped from public exam papers that are already inside training corpora, by construction.

Translated

MMLU clones in Indic packaging measure translation transfer — not native competence, not India.

Closed

The one frontier-grade effort (OpenAI's IndQA) can't be run, audited, or built upon by anyone else.

इ IndiBench
03
Why now

Models already serve hundreds of millions of Indians. Nobody is keeping score.

  • Voice is the interface for India's next users — and ASR still runs 20–30% word-error-rate on Indian languages.
  • IndQA (Nov 2025) proved the demand — then kept the data, grading, and pipeline closed. The open, lab-neutral slot is unclaimed.
  • India AI Summit (Feb 2026): frontier labs committed to benchmark and dataset support for under-represented languages. The window is open.
  • Every static benchmark decays. Ours regenerates — a renewable benchmark stays hard while the ecosystem's snapshots go stale.
इ IndiBench
04
Solution

The pipeline is the product.

Questions are synthesized from hard Indian source documents, then adversarially filtered: an item survives only if ≥2 of 4 frontier models fail it — one each from OpenAI, Anthropic, Google, plus the best Indian model.

S0
Ground
license-reviewed textbooks, gazettes, case law
S1
Author
frontier LLM writes natively in-script; key comes from the source
S2
Verify
independent panel re-derives every answer key
S3
Filter
cross-lab panel answers blind — keep only what ≥2/4 fail
S4
Audit
native experts spot-check; safety items 100%
S5
Release
canary + private split + 6-monthly regeneration

Contamination defense: canary GUID · never-published held-out split · renewable refresh waves

The ask

Be the yardstick, not a yardstick.

  • Frontier labs: panel + judge API credits — your models grade nothing alone, and every log is published.
  • AI4Bharat · BharatGen · IndiaAI: distribution, source corpora, and the Indic LLM-Arena partnership.
  • Native-language domain experts: spot-check items, earn co-authorship on the paper.
  • Everyone else: git clone github.com/pukrvi/indibench — the whole pipeline runs offline today.
